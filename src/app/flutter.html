üöÄ Getting Started with Flutter

Flutter is Google's UI toolkit for building beautiful, natively compiled applications for mobile, web, and desktop from a single codebase.

Prerequisites

  • Install Flutter SDK from flutter.dev
  • Install an IDE (VS Code or Android Studio)
  • Set up an emulator or physical device

Creating Your First App

Open your terminal and run:

# Create a new Flutter project flutter create my_first_app # Navigate to the project cd my_first_app # Run the app flutter run

Understanding main.dart

Every Flutter app starts with main.dart. Here's the basic structure:

import 'package:flutter/material.dart'; void main() { runApp(MyApp()); } class MyApp extends StatelessWidget { @override Widget build(BuildContext context) { return MaterialApp( title: 'My First App', home: HomePage(), ); } }
üí° Tip: The main() function is the entry point, and runApp() inflates the given widget and attaches it to the screen.

üß© Understanding Widgets

In Flutter, everything is a widget! Widgets are the building blocks of your UI.

Two Types of Widgets

  • StatelessWidget: Immutable widgets that don't change
  • StatefulWidget: Widgets that can change state over time

StatelessWidget Example

class GreetingWidget extends StatelessWidget { final String name; GreetingWidget({required this.name}); @override Widget build(BuildContext context) { return Text( 'Hello, $name!', style: TextStyle(fontSize: 24), ); } }

StatefulWidget Example

class CounterWidget extends StatefulWidget { @override _CounterWidgetState createState() => _CounterWidgetState(); } class _CounterWidgetState extends State<CounterWidget> { int counter = 0; void _incrementCounter() { setState(() { counter++; }); } @override Widget build(BuildContext context) { return Column( children: [ Text('Count: $counter'), ElevatedButton( onPressed: _incrementCounter, child: Text('Increment'), ), ], ); } }

üß≠ Navigation

Flutter uses a Navigator to manage a stack of Route objects for screen transitions.

Basic Navigation

// Navigate to a new screen Navigator.push( context, MaterialPageRoute(builder: (context) => SecondScreen()), ); // Go back to previous screen Navigator.pop(context);

Named Routes

For larger apps, use named routes for better organization:

// Define routes in MaterialApp MaterialApp( initialRoute: '/', routes: { '/': (context) => HomeScreen(), '/details': (context) => DetailsScreen(), '/settings': (context) => SettingsScreen(), }, ) // Navigate using named routes Navigator.pushNamed(context, '/details');

Passing Data Between Screens

// Send data to next screen Navigator.push( context, MaterialPageRoute( builder: (context) => DetailScreen( title: 'Item 1', description: 'Details about item', ), ), ); // Receive data in the screen class DetailScreen extends StatelessWidget { final String title; final String description; DetailScreen({required this.title, required this.description}); @override Widget build(BuildContext context) { return Scaffold( appBar: AppBar(title: Text(title)), body: Text(description), ); } }

Basic Theme Setup

class MyApp extends StatefulWidget { @override _MyAppState createState() => _MyAppState(); } class _MyAppState extends State<MyApp> { ThemeMode _themeMode = ThemeMode.light; void _toggleTheme() { setState(() { _themeMode = _themeMode == ThemeMode.light ? ThemeMode.dark : ThemeMode.light; }); } @override Widget build(BuildContext context) { return MaterialApp( themeMode: _themeMode, theme: ThemeData.light(), darkTheme: ThemeData.dark(), home: HomePage(onToggleTheme: _toggleTheme), ); } }

üíæ Persistent Storage

Store data locally on the device using SharedPreferences for simple key-value pairs.

Add Dependency

Add to pubspec.yaml:

dependencies: shared_preferences: ^2.2.0

Saving Data

import 'package:shared_preferences/shared_preferences.dart'; // Save data Future<void> saveUserData() async { final prefs = await SharedPreferences.getInstance(); await prefs.setString('username', 'JohnDoe'); await prefs.setInt('age', 25); await prefs.setBool('isDarkMode', true); await prefs.setDouble('score', 98.5); // Save list await prefs.setStringList('favorites', ['item1', 'item2']); }

Loading Data

// Load data Future<void> loadUserData() async { final prefs = await SharedPreferences.getInstance(); String? username = prefs.getString('username'); int? age = prefs.getInt('age'); bool? isDarkMode = prefs.getBool('isDarkMode') ?? false; double? score = prefs.getDouble('score'); List<String>? favorites = prefs.getStringList('favorites'); }

Complete Example

class SettingsScreen extends StatefulWidget { @override _SettingsScreenState createState() => _SettingsScreenState(); } class _SettingsScreenState extends State<SettingsScreen> { bool _notifications = false; @override void initState() { super.initState(); _loadSettings(); } Future<void> _loadSettings() async { final prefs = await SharedPreferences.getInstance(); setState(() { _notifications = prefs.getBool('notifications') ?? false; }); } Future<void> _saveSettings() async { final prefs = await SharedPreferences.getInstance(); await prefs.setBool('notifications', _notifications); } @override Widget build(BuildContext context) { return SwitchListTile( title: Text('Enable Notifications'), value: _notifications, onChanged: (value) { setState(() { _notifications = value; }); _saveSettings(); }, ); } }

üì¶ Understanding pubspec.yaml

The pubspec.yaml file is the heart of your Flutter project. It defines dependencies, assets, and metadata.

Basic Structure

name: my_app description: A new Flutter project. publish_to: 'none' version: 1.0.0+1 environment: sdk: '>=3.0.0 <4.0.0' dependencies: flutter: sdk: flutter cupertino_icons: ^1.0.2 http: ^1.1.0 shared_preferences: ^2.2.0 dev_dependencies: flutter_test: sdk: flutter flutter_lints: ^2.0.0 flutter: uses-material-design: true assets: - assets/images/ - assets/data.json fonts: - family: Roboto fonts: - asset: assets/fonts/Roboto-Regular.ttf - asset: assets/fonts/Roboto-Bold.ttf weight: 700

Key Sections Explained

  • name: Your app's package name (lowercase, underscores)
  • version: Format is major.minor.patch+buildNumber
  • environment: Dart SDK version constraints
  • dependencies: Packages your app uses at runtime
  • dev_dependencies: Packages for development only

Adding Dependencies

# Method 1: Add to pubspec.yaml manually dependencies: package_name: ^1.0.0 # Method 2: Use command line flutter pub add package_name # After adding, run: flutter pub get

Version Constraints

# Caret syntax (recommended) - compatible updates http: ^1.1.0 # Allows 1.1.0 to <2.0.0 # Exact version http: 1.1.0 # Range http: '>=1.0.0 <2.0.0' # Any version http: any
üí° Best Practice: Use caret syntax (^) for automatic compatible updates. Run flutter pub outdated to check for updates.

üêõ Console Logging & Debugging

Learn how to debug your Flutter apps effectively using console logging.

Basic Print Statements

// Simple print print('Hello from Flutter!'); // Print variables int counter = 5; print('Counter value: $counter'); // Print objects List<String> items = ['Apple', 'Banana']; print('Items: $items');

debugPrint (Recommended)

import 'package:flutter/foundation.dart'; // Better for Flutter - handles large outputs debugPrint('This is a debug message'); // Automatically removed in release builds if (kDebugMode) { debugPrint('Debug-only message'); }

Developer Logger

import 'dart:developer' as developer; // More structured logging developer.log( 'User logged in', name: 'AuthService', error: errorObject, stackTrace: stackTrace, ); // With custom levels developer.log( 'API call completed', name: 'NetworkService', level: 800, // INFO level );

Logging Best Practices

class ApiService { Future<void> fetchData() async { try { debugPrint('[API] Fetching data...'); // API call final response = await http.get(url); debugPrint('[API] Response: ${response.statusCode}'); debugPrint('[API] Body length: ${response.body.length}'); } catch (e, stackTrace) { developer.log( 'Failed to fetch data', name: 'ApiService', error: e, stackTrace: stackTrace, ); } } }

Common Debugging Scenarios

// 1. Widget lifecycle @override void initState() { super.initState(); debugPrint('[Widget] initState called'); } // 2. Button press tracking void onButtonPressed() { debugPrint('[UI] Button pressed at ${DateTime.now()}'); // action code } // 3. Navigation tracking Navigator.push(context, route).then((_) { debugPrint('[Navigation] Returned from DetailScreen'); }); // 4. State changes setState(() { counter++; debugPrint('[State] Counter updated to $counter'); });
