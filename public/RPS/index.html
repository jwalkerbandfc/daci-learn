Step 6

Smarter Win Logic

Right now we write a separate elif for every win. That's three lines just for victories—and if we ever added "lizard" and "spock", it would explode into dozens! Let's teach Python the rules once using a dictionary.

💡 The Big Idea: "What beats what?"

Every move beats exactly one other move. Rock beats scissors, paper beats rock, scissors beats paper. A dictionary stores these pairs as key: value links. We can then ask one simple question: "Does the player's move beat the computer's move?" No need to list every combination by hand.

📝 Replace your whole if/elif/else block with this: 
# 3. Determine winner using a "beats" dictionary
# Each move maps to the move it defeats:
beats = {
    "rock": "scissors",
    "paper": "rock",
    "scissors": "paper"
}

if player_choice == computer_choice:
    print("It's a draw! Play again.")
elif beats[player_choice] == computer_choice:
    print("You win!")
else:
    print("The computer wins! Better luck next time.")

The Next Missing Link

There's still one annoyance: the game plays once and quits. To play again you have to re-run the whole program! Next we'll wrap everything in a while loop so you can play round after round and keep score.

Step 7

Loop the Game with while

A while loop repeats a block of code as long as a condition stays true. We'll use it to let the player keep playing rounds until they type quit—while quietly tracking the score.

🔁 Three loop keywords you'll meet

  • while True: — repeat forever, until we deliberately stop.
  • break — jump out of the loop immediately (we use it for "quit").
  • continue — skip the rest of this round and start the next one (handy for invalid input).
📝 Replace main.py with this looping version: 
import random

game_options = ["rock", "paper", "scissors"]
beats = {"rock": "scissors", "paper": "rock", "scissors": "paper"}

player_score = 0
computer_score = 0

print("=== ROCK, PAPER, SCISSORS ===")
print("Type 'quit' at any time to stop.\n")

# Keep playing rounds until the player chooses to quit
while True:
    player_choice = input("Enter rock, paper, or scissors: ")

    if player_choice == "quit":
        break

    if player_choice not in game_options:
        print("That's not a valid move. Try again!\n")
        continue

    computer_choice = random.choice(game_options)
    print("Computer chose: " + computer_choice)

    if player_choice == computer_choice:
        print("It's a draw!")
    elif beats[player_choice] == computer_choice:
        player_score += 1
        print("You win this round!")
    else:
        computer_score += 1
        print("The computer wins this round!")

    print(f"Score -> You: {player_score} | Computer: {computer_score}\n")

print("\nThanks for playing!")
print(f"Final score -> You: {player_score} | Computer: {computer_score}")

Step 8

Build a GUI with Tkinter

tkinter ships with Python—nothing to install. A GUI (Graphical User Interface) swaps typed commands for windows, labels, and clickable buttons. We'll learn the building blocks first, then assemble a fully clickable game.

🧱 The three pieces of every Tkinter app

The Window

tk.Tk() creates the main window that holds everything.

Widgets

Label shows text, Button is clickable. .pack() places them.

The Loop

window.mainloop() keeps the window open and listening for clicks.

🪟 Example 1 — Your first window 
import tkinter as tk

window = tk.Tk()
window.title("My First Window")
window.geometry("300x200")   # width x height in pixels

window.mainloop()

Run this and a small empty window appears. Close it to end the program. That's a complete GUI app in six lines!

🔘 Example 2 — A label and a button that does something 
import tkinter as tk

# A function that runs when the button is clicked
def say_hello():
    label.config(text="Hello! You clicked the button.")

window = tk.Tk()
window.title("Buttons and Labels")
window.geometry("320x160")

label = tk.Label(window, text="Click the button below")
label.pack(pady=20)

button = tk.Button(window, text="Click Me", command=say_hello)
button.pack()

window.mainloop()

Notice command=say_hello has no brackets. We hand Tkinter the function itself, and Tkinter calls it later, each time the button is clicked. label.config(...) updates the label's text on the fly.

🎮 Example 3 — Rock Paper Scissors as a clickable GUI 
import tkinter as tk
import random

options = ["rock", "paper", "scissors"]
beats = {"rock": "scissors", "paper": "rock", "scissors": "paper"}

# This runs whenever the player clicks one of the three buttons
def play(player_choice):
    computer_choice = random.choice(options)

    if player_choice == computer_choice:
        result = "It's a draw!"
    elif beats[player_choice] == computer_choice:
        result = "You win!"
    else:
        result = "Computer wins!"

    result_label.config(
        text=f"You chose: {player_choice}\n"
             f"Computer chose: {computer_choice}\n\n{result}"
    )

window = tk.Tk()
window.title("Rock Paper Scissors")
window.geometry("340x260")

title = tk.Label(window, text="Rock, Paper, Scissors", font=("Arial", 16, "bold"))
title.pack(pady=10)

# A frame to hold the three buttons side by side
button_frame = tk.Frame(window)
button_frame.pack(pady=10)

rock_btn = tk.Button(button_frame, text="Rock", width=8,
                     command=lambda: play("rock"))
paper_btn = tk.Button(button_frame, text="Paper", width=8,
                      command=lambda: play("paper"))
scissors_btn = tk.Button(button_frame, text="Scissors", width=8,
                         command=lambda: play("scissors"))

rock_btn.pack(side="left", padx=4)
paper_btn.pack(side="left", padx=4)
scissors_btn.pack(side="left", padx=4)

result_label = tk.Label(window, text="Make your move!", font=("Arial", 12))
result_label.pack(pady=20)

window.mainloop()

🔍 Why lambda here?

A button's command wants a function with no brackets, but we need to pass which move was picked. lambda: play("rock") creates a tiny throwaway function that calls play("rock") for us when clicked. Same game logic as before—the dictionary does all the deciding—just wired to buttons instead of input().
